# Demo guide

This doc covers how to demo SNAP Notice Navigator reliably (including for a recorded video), the seeded
sample notices, and how "demo mode" works.

## Demo mode (reliable, no live failures)

The app is built so a demo never breaks mid-flow:

- **Seeded sample notices.** The upload page offers three synthetic sample notices. Selecting one skips live
  file handling entirely and feeds known-good text into the pipeline — no OCR risk, no upload risk.
- **Offline-safe AI.** If `ANTHROPIC_API_KEY` is not set (or the network/LLM call fails), every AI step falls
  back to the deterministic rules + templates path and still produces a complete, source-backed result.

**Two ways to run the demo:**

| Mode | How | What it shows |
|---|---|---|
| **Offline (zero setup)** | No `.env.local`. Just `npm run dev`. | The full journey on the rules + curated-template path. Classification, field extraction, retrieval, and source-grounded recommendations all work. Most reliable for a recording. |
| **Full AI** | Put `ANTHROPIC_API_KEY` in `.env.local`. | The same flow, but the explanation/steps are generated by Claude (`claude-sonnet-4-6`) grounded in the retrieved sources, and the results page shows "Guidance generated from official sources." |

> Tip for recordings: pre-run one sample once with your API key so the structured-output schema is cached and
> the first real call is fast. For maximum reliability, demo offline.

## The 5-minute demo script

1. **Landing** — read the one-liner and the "Who decides what" strip (AI explains → you decide → agency decides). Click **Try a sample notice**.
2. **Upload** — pick the **Denial notice** sample. (Mention: real users upload a PDF/photo or paste text.)
3. **Review** — point out this is the **human-in-the-loop gate**: the user sees exactly what was read, can edit it, and must check *"I've read this and it matches"* before anything is analyzed.
4. **Questions** — answer the short intake (state = California, situation = appealing). Note it's small and calming.
5. **Results** — walk the hierarchy:
   - status card (notice type + confidence),
   - plain-language explanation and likely reason,
   - the **prioritized next-steps** card (framed as options),
   - documents to gather,
   - the **Official sources** card (real USDA + CalFresh links),
   - the **Need more help** card (211, legal aid),
   - expand **"See how we read your notice"** to show the AI reasoning + confidence meter + extracted fields,
   - the persistent **"your agency decides"** disclaimer.
6. **Show the safeguards** — try the **Recertification** sample (lower confidence) to show the supportive
   low-confidence card, or pick an unsupported state to show the federal-baseline fallback banner.

## Seeded sample notices

Defined in `src/data/cases.ts` (synthetic — no real personal data). Each is wired to a sensible default state/situation.

| Sample | Notice type | State | Demonstrates |
|---|---|---|---|
| **Denial notice** | `denial` | CA (CalFresh) | Income-over-limit denial; appeal/hearing within 90 days; reapply option. High-confidence, full extraction. |
| **Request for verification** | `verification_request` | TX (HHSC, Form H1020) | Document request with a hard due date; "send docs, don't panic" framing. |
| **Recertification issue** | `recertification` | MO (Family Support Division) | Renewal not finished before benefits end; interview + form steps; **lower confidence → supportive fallback card.** |

## Verifying the pipeline from the terminal

With the dev server running (`npm run dev`):

```bash
# Classification + field extraction (Phase 3)
curl -s -X POST http://localhost:3000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":"CalFresh Notice of Action. Your application has been DENIED. Reason: income over the gross income limit. You may request a state hearing within 90 days."}'

# Full pipeline: classify + retrieve + recommend (Phase 3 + 4)
curl -s -X POST http://localhost:3000/api/recommend \
  -H "Content-Type: application/json" \
  -d '{"text":"CalFresh Notice of Action. Your application has been DENIED. Reason: income over the gross income limit.","intake":{"state":"CA","actionContext":"appealing"}}'
```

A supported state (CA/TX/MO) returns state + federal sources and `stateSupported: true`; any other state returns
federal-only sources and `stateSupported: false`.
